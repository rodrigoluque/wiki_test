Goal

The reason is to inform the changes on the API to consult the events of packages in our systems.

General Description

The following document shows the properties of the new API to use and a new sub-event who shows more details of the state of the package 

Properties of new API:

The new API has a new structure much simpler:

  • The new API returns all the events and sub-events of the package that you are consulting
  • The new API only returns information of events and sub-events
  • The new API allows consulting for more than one tracking number or sales order number at the same time

Response fields:

This is a new structure of the Response:

Fields:

  • tracking: Package tracking number,
  • origin: Package origin country,
  • destination": Package destination country,
  • state: State of packages destination country,
  • events:  List of events "
[
Date: date of event
Code: code of event. This code is unique
Description: Description of sub-event. This field will shows a more specific information of the state of the package
Description_alt: Detailed description of event 
Category: Category of the event
Received_by: Information of person that received the package 
Office: office where is the package
City: City where is the package
]

"

  • Example of use the API for a one tracking number 
    • URL:
      • http://tracking.mailamericas.com/api/v2/events?tracking=tracking_number&access_token=access_token
    • Response:
  • Example of using the API for more than one tracking number
    • URL:
      • http://tracking.mailamericas.com/api/v2/events?tracking=tracking_number_1,tracking_number_2,tracking_number_3,tracking_number_X&access_token=access_token
  • Example of use the API for a one sales order number 
    • URL:
      • http://tracking.mailamericas.com/api/v2/events? sales_order_number= sales order number&access_token=access_token
  • Example of using the API for more than one sales order number
    • URL:
      • http://tracking.mailamericas.com/api/v2/events? sales_order_number= sales order number _1, sales order number _2, sales order number _3, sales order number _X&access_token=access_token
  • NOTE:
    • The API always returns the descriptions in ENGLISH, except for the "description_alt" field which is sent in the same language in which it is received from our suppliers
